# 1. 자바스크립트란

- **동적인 웹 사이트 제작을 위해 사용**됨

1. 역사 : 1990년대 넷스케이프의 브랜던 아이크(Brenden Eich)가 Mocha라는 이름으로 출시하고, 라이브스크립트로 개발하였다가 최종적으로 자바스크립트가 됨. 그러나 브라우져(넷스케이프, 익스플로러....)에 따라 상이한 스크립트가 나오기 시작하면서 개발이 쉽지 않았으며(if 구문의 남발...).... 스크립트의 표준화에 대한 요구가 생기기 시작함.

2. ECMA(European Computer Manufactures Association) 스크립트 : 자바스크립트의 표준작업이 시작된 것으로 1997년 6월 정식으로 채택된 스크립트의 표준. 모던 스크립트는 ES(ECMA script의 약자)6 버전에서 시작한다고 본다. + 이번 2026년에 ES2026이 생긴다

3. 바닐라 자바스크립트(바닐라 JS) : 어떠한 라이브러리(JQeury..)나 프레임워크(React, Vue, Angular....)를 사용하지 않는 순수 자바스크립트, 가장 빠르고 가벼운 크로스 플랫폼 프레임워크 => 실력 향상을 위한 필수 단계 => 결코 쉽지 않다...=> 그러나 프레임워크나 라이브러리는 모두 바닐라 자바스크립트로 만들어져 있다. => 자신의 실력과 가치를 높이는 강력한 수단이 될 수 있다.

4. 바닐라 자바스크립트 시작하기

- 사이트 : http://vanilla-js.com/
- 시작 : <script src="path/to/vanilla.js"></script>를 html 문서안에 삽입
- 위트 있는 사이트 ^^ : 다운받아도 아무 것도 없다!!!!
- 순수 자바스크립트로 짜면 좋다!!!!

- 최근에는 자바스크립트를 이용하여 웹뿐만 아니라 서버(NodeJS), 데스크톱 프로그램(Electron)도 제작하고 있는 추세
- 최근 [가장 인기있는 언어](http://githut.info/)로 높은 점유율을 차지하고 있음

# 4. 변수 선언자 var, let, const

- 변수란?
  : 프로그램 실행에 필요한 데이터를 담는 기억공간
  - 장점
    : 데이터 타입에 따른 선언이 필요하지 않다
  - 단점
    : 데이터 타입에 관계없는 변수이기 때문에 개발 시 오류 발생이 가능

- 변수 선언자 종류
  : var, let, const

- 선언 형식
  : 선언자 변수명 = 값;

- 변수 작성 규칙(4가지) : 카멜, 스네이크, 파스칼, 헝가리안

      - 카멜 표기법 (userName) <--- 자바스크립트는 카멜 표기법을 따름
         : 두 가지 이상의 단어가 합쳐질 경우, 두번째부터 단어의 첫 글자를 대문자로
      - 스네이크 표기법 (user_name)
         : 두개 이상의 단어가 합쳐질 경우 단어 사이에 _ 삽입
      - 파스칼 표기법 (UserName)
          : 모든 단어의 첫 글자를 대문자로
      - 헝가리안 표기법 (struserName)
         : 변수명 맨 앞에 데이터의 타입에 해당하는 영어 단어를 추가하고, 나머지는 카멜 표기법을 따른다

      + 최근 추세는 1번!!!!! 카멜 케이스!!!!!!(이런거잇다는건알고잇스라공~)

- var 선언자의 특징 - Es6 이전부터 있었음 - 재선언 (동일한 이름으로) 가능, 재할당 가능 - 전역변수로 사용 가능 - 덮어쓰기가 가능해서 > 오류의 가능성을 늘 가지고 있음 > 항상 조심조심해서 사용

- **let** 선언자의 특징
  - Es6 이후에 등장
  - 재선언 불가 > 프로그램 안에서 동일한 변수명 사용 불가능 > 협업 때는 let을 권장함!
  - 재할당 가능

- **const** 선언자의 특징
  - Es6 이후에 등장
  - 상속값(변하지 않는)
  - 가급적 대문자로 선언(자바와 같음)
  - 상수니 > 재할당, 재선언 불가! - 협업 => 팀 => 포맷 지정시 활용
  - 협업하는 개발자들이 통일해서 사용하는 데이토 선언 등에 사용

# 10. 조건문

- 특징
  - 가장 중요한 구문 중 하나이다.
  - 프로그램 = 조건문의 집합
  - if : 만일 ~라면...
  - 작성 시 순서가 "효율성"에 영향을 미친다.

1. 조건문 if

- if :
  if (조건식) {
  조건을 만족할 경우의 코드 블럭
  }
- if else :
  if else (조건문(if 조건문에 해당하지 않은 범위)){
  조건에 만족할 경우의 코드 블럭
  }
- if else if else if .... else :
  if (조건문1) {
  조건문1을 만족하는 경우 코드 블럭
  }
  else if(조건문2) {
  조건문1을 만족하지 않고 조건식 2를 만족하는 경우 코드 블럭
  }
  else if(조건문3) {
  조건문 1,2에 만족하지 않고 조건문 3에 만족하는 경우 코드 블럭
  }
  else {
  조건문 1,2,3을 모두 만족하지 않을 경우 코드 블럭
  }
- 항상 false 인 값 :
- 실습(09_if.html)

2. 조건문 switch

- 특징
  - 상수만 비교
  - 실무에서 거의 사용하지 않는다
- 장점
  : if문을 사용하면 모든 조건을 비교하면서 내려오지만, switch문의 경우, 값을 비교해서 바로 찾아간다.
  즉, 성능 향상 장점이 있다. => 컴퓨터의 성능이 워낙 좋으므로 크게 신경쓰지 않는다

# 11. 반복문

- 특징

1. for loop

- 특징 : 배열에서 가장 많이 사용
- 동작 원리 :
- for (statement1 ; statement2; statement3){

        조건절이 참인 동안 실행할 코드


        //statement1 : 코드 블록을 실행하기 전에 무조건 한 번 실행하는 초기화 문장

        //statement2 : 코드 블록을 실행시킬 조건절

        //statement3:  코드 블록을 실행한 후 실행되는 횟수 증가 코드

  }

- for 과 break : break 를 만나면 반복문 블럭을 중단하고 블록을 빠져나온다.
- for 과 continue : continue 를 만나면, 그 단계는 진행하지 않고, 다음 step 으로 넘어간다.

1-1. for ~ in

- 특징
- 배열, object 에서 많이 사용(순차적으로 순회할 수 있는 자료 구조들에서 많이 사용)

        let brands = ["애플", "구글", "아마존", "마이크로소프트", "메타"];

        for (const i in brands) {
            //i는 인덱스
            console.log(i);
